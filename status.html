What the numbers mean

  • visits (local) — total page loads tracked in your browser's localStorage. Each visitor sees their own stats only.
  • today — visits since midnight (your local time).
  • avg time — average time spent on this page across your sessions.
  • pages — unique page paths you've visited on this site.
  • clicks tracked — total clicks on links, buttons, and other elements recorded from your browser.
  • most visited pages — which URLs you viewed most, sorted by count with a bar chart.
  • most clicked elements — which buttons / links you interacted with most.
  • recent visits — chronological log of your page views with referrer info.

All this data lives only in your browser's localStorage. It never leaves your machine. Clear it anytime with the "clear local" button.

GitHub Token — what it's for

This field stores a GitHub personal access token in your browser's localStorage so a future script can call the GitHub API (e.g., fetch repo info, stars, commits, issues). It is not used by anything yet — it's just a safe place to keep it.

The token is never sent anywhere unless you explicitly build a feature that uses it. It stays in your browser, on your machine.

What kind of token to use

  • Go to github.com/settings/tokens
  • Generate a fine-grained PAT or classic token
  • For read-only public data, select scope public_repo (or metadata:read)
  • No expiration is fine if this is your personal utility page. For extra safety, use a fine-grained token limited to this repo only.
  • Paste it into the field above and click save — it's stored in your localStorage.

⚠️ This page is noindex, but anyone with the URL can open it. Treat your token accordingly.

Server-side intelligence

Below the analytics section, there's a collapsible block labelled "server-side intelligence" fetched from the Vercel serverless function (/api/status). It shows your visitor data (IP, country, city, browser, language), deployment info (environment, region), and git metadata (commit SHA, branch, message, repo). This data comes from Vercel's request headers — nothing is stored server-side.
